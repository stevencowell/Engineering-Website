Project Brief

Design, build and test a small garden watering system that uses an input, a control process and an output to water a plant area safely and efficiently.

  • Input: moisture sensor, rain sensor, switch, timer or teacher-approved trigger.
  • Process: controller, relay, transistor circuit, timer or simple logic.
  • Output: pump, solenoid valve, indicator or water outlet.
  • Evidence: diagrams, algorithm, build log, test data, fault log and evaluation.

System Overview

Student Checklist

  • Define the watering problem and success criteria.
  • Draw the input-process-output system.
  • Create an algorithm or flowchart.
  • Plan safe wet and dry zones.
  • Build and test one subsystem at a time.
  • Record faults and improvements as they happen.

Build Steps

  1. Confirm the brief, success criteria and approved component list.
  2. Sketch the system layout and mark wet/dry zones.
  3. Build the water path and test for leaks without electronics connected.
  4. Build the input and output circuits separately.
  5. Connect the control process after teacher checking.
  6. Run repeated tests and record results.
  7. Modify one issue at a time and update the fault log.

Bench Checklist

  • Wet zone and dry electronics zone are clearly separated.
  • Power is disconnected while wiring is changed.
  • Tubing is secure and water path is visible.
  • Teacher has checked the circuit before powered testing.
  • Photos and test data are captured for the folio.
