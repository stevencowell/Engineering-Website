Project

Learning Focus

This toolkit brings the project together: build the prototype safely, test output in a controlled way, improve one or two design features, and use evidence to explain what worked.

Build And Test Sequence

  1. Check the final design and materials before cutting or drilling.
  2. Build the frame square and stable.
  3. Fit the shaft, impellor and supports so the rotating parts spin freely.
  4. Mount the generator and USB unit securely.
  5. Complete a teacher safety check before powered output testing.
  6. Test output, record data, photograph evidence and identify faults.
  7. Make controlled improvements and retest using the same method.
Folio evidence: Include build photos, a test table, fault-finding notes, improvement evidence and a final evaluation paragraph using test data.

Risk Controls

Before Construction Starts

A good build starts with a checked design. The charger has small rotating parts, a small generator shaft and electrical output, so rushing into construction usually creates avoidable problems.

Teacher Check 1: Build Readiness

  • Final design fits inside the size limit.
  • Generator, shaft and impellor positions are shown.
  • Materials and fixings are selected.
  • WHS controls are identified.
  • Construction sequence is sensible.

Teacher Check 2: Test Readiness

  • Frame is stable on the bench.
  • Rotating parts clear the frame and wires.
  • Generator and USB module are mounted securely.
  • Electrical connections are checked.
  • Test area and stop method are clear.

Build Quality

Build quality affects performance. A charger with a clever blade design can still fail if the frame is crooked, the shaft rubs, the generator twists or the USB module hangs loose.

Build featureGood evidenceProblem sign
FrameSquare, stable, compact and strong enough for testing.Rocks on the bench, flexes under load or falls outside the size limit.
ImpellorBalanced, secure on the shaft and clear of all supports.Wobbles, scrapes, throws air unevenly or slips on the hub.
Shaft supportRotates freely with little side movement.Binds, bends or vibrates at higher speed.
Generator mountHeld firmly and aligned with the shaft or gear train.Moves during testing or pulls wires tight.
Electrical outputConnections secure, polarity checked and USB module protected.Loose wires, intermittent readings or unsafe device connection.

Testing Method

Testing must be fair enough to compare changes. If students change the wind source, blade angle, distance and gear setup all at once, the results will be a mess. One change at a time. Boring? Yes. Useful? Also yes.

  1. Set the baseline: record the first working setup, including blade design, drive method, wind source and distance.
  2. Inspect safety: check frame stability, rotating clearances, loose wires, teacher sign-off and stop method.
  3. Run a short spin test: observe starting behaviour, wobble, rubbing and noise before measuring output.
  4. Measure output: record voltage or approved output readings if equipment is available.
  5. Change one variable: blade angle, blade number, gear ratio, frame support or wind distance.
  6. Retest under the same conditions: compare the new result with the baseline.
  7. Record the judgement: better, worse or no clear improvement, with evidence.
Testing rule: If the method cannot be repeated, the data cannot be trusted.

Writing The Final Evaluation

A strong evaluation links evidence, engineering theory and improvement. It should sound like an engineer reviewing a prototype, not a student trying to survive the last five minutes of the lesson.

Sentence frame: My final design was successful/partly successful because the test data showed _____. The main problem was _____, which was caused by _____. I improved this by _____. If I built a second version, I would _____ because _____.

Include these points

  • Best output or best performance recorded.
  • Most important design feature affecting performance.
  • Faults found during testing and how they were fixed.
  • How well the model met the size, safety and function requirements.
  • Material and sustainability judgement.
  • Specific second-version improvement.

Activities And Folio Evidence

1. Build Log

  • Record each major build stage with a photo and short caption.
  • Explain one problem or decision from each stage.

2. Safety Sign-Off

  • Include teacher check notes before powered testing.
  • Record rotating-part and electrical-output controls.

3. Test Table And Fault Log

  • Use a repeatable test method and record results.
  • Keep faults visible in the folio. Problems are evidence if you respond to them properly.

4. Final Evaluation

  • Write a final judgement using data, photos, WHS, sustainability and design improvement language.
  • Answer the folio prompts: what went well, what did not go as planned, and what would you change?

Common Mistakes To Avoid

  • Testing without a baseline: you need an original result to compare improvements against.
  • Changing everything at once: this destroys cause-and-effect evidence.
  • Only photographing the final model: build-process photos are part of the evidence.
  • Hiding failures: fault-finding is engineering evidence, not embarrassment.
  • Evaluation without data: a strong judgement must refer to observations, test results or photos.

Toolkit Self-Check

Why must the charger have a teacher safety check before powered output testing?

A teacher check confirms that the frame is stable, rotating parts are clear, wiring is secure, polarity is correct and the output will be tested with approved equipment. It reduces the chance of damage, injury or unreliable data.

Why should output testing be done in controlled steps instead of one full-speed test?

Controlled steps let students observe wobble, vibration, loose parts, voltage changes and wiring faults before the risk increases. Small steps also produce better data because each change can be linked to the result.

What should you do if the impellor wobbles, rubs or vibrates during testing?

Stop the test, move clear of the spinning parts, power down or remove wind input if safe, and tell the teacher. Then inspect blade balance, shaft support, frame alignment and loose fixings before retesting.

What data should be recorded to evaluate charger performance properly?

Useful data includes test conditions, wind source or speed setting, voltage or output reading, blade design, gear setup, faults observed, changes made and retest results. Photos or short notes help prove what happened.

Why should only one design feature be changed before retesting?

Changing one feature at a time makes the test fair. If blade angle, gearing and generator position all change together, students cannot tell which change improved or reduced performance.

How well did the final model meet the size, safety and function requirements?

A strong answer uses evidence. It states whether the model stayed within 250 x 150 x 100 mm, whether it operated safely, whether it produced useful output, and what test data supports that judgement.

What evidence should appear in the folio for build, test and evaluate?

The folio should include build photos, teacher check notes, a test table, fault-finding notes, improvement evidence, retest results and a final evaluation paragraph that uses data rather than just saying it worked.

What would be a worthwhile change if you built a second version of the charger?

A worthwhile change is linked to evidence. Examples include improving blade pitch, reducing friction, strengthening the frame, changing gear ratio, protecting wiring or improving the test setup because the first version showed a specific weakness.
