Goals for this activity

In this task you will be expected to:

  • Apply safe work practices when using low-voltage electronics, water, tools and testing equipment.
  • Explain the system using input, process, output and feedback language.
  • Design and build a working watering prototype using appropriate components.
  • Use an algorithm, flowchart or pseudocode to explain the watering decision.
  • Test the system using repeatable data and record faults, fixes and improvements.
  • Evaluate reliability, water efficiency, sustainability, construction quality and possible improvements.

Task Outline

This assessment task is based on your Semester 1 work on the Garden Watering System project. You will investigate a watering problem, design a small control system, build a working prototype, test it, and document the complete process in an engineering folio.

Garden Watering System Project - Semester 1

  • Project: Design, build and test a small automated or semi-automated garden watering system.
  • Core requirement: The system must include an input, a control process and an output.
  • Possible inputs: moisture sensor, rain sensor, switch, timer or teacher-approved trigger.
  • Possible outputs: pump, solenoid valve, LED indicator, buzzer or water delivery outlet.
  • Timing:
    • Weeks 1-4: WHS, control systems overview, irrigation problem, project brief and safe test setup
    • Weeks 5-8: inputs, outputs, sensors, relays, algorithms and circuit planning
    • Weeks 9-12: design development, component selection, wiring/circuit planning, irrigation layout and build approval
    • Weeks 13-16: prototype build, subsystem tests, integration and fault-finding
    • Weeks 17-20: reliability testing, improvements, folio completion, evaluation and submission
  • Evidence required:
    • Design brief analysis and success criteria.
    • WHS risk controls for electronics, water, tools, pump/valve setup and testing.
    • Input-process-output diagram and system block diagram.
    • Algorithm, flowchart or pseudocode explaining when the system waters.
    • Annotated concept sketches and final design choice.
    • Wiring/circuit plan and irrigation layout.
    • Materials/component list and build sequence.
    • Build log with photos showing key stages.
    • Test data including trigger condition, response time, water delivered, faults and result.
    • Fault log showing issue, likely cause, fix attempted and whether the fix worked.
    • Final evaluation using evidence from testing.

Criteria

In this task you will be assessed on how well you:

  • Apply WHS practices and manage risks while working with water, low-voltage electronics and tools.
  • Develop a control-system solution that responds to a watering condition.
  • Select and use appropriate components, materials, tools and processes.
  • Communicate the design clearly using diagrams, sketches, flowcharts and folio writing.
  • Test the system methodically and record useful data.
  • Evaluate the system's function, reliability, water efficiency and sustainability.
